Système multi-agents

  • Une distribution d'agents autonomes qui coopèrent ensemble pour atteindre le(s) but(s) du système.
  • La solution du problème émerge de l'interaction et du comportement des agents.

Modélisation des systèmes multi-agents

Pricipalement deux approches :

  • Approche centrée sur l'agent (ACMAS, Agent Centred Multi-Agent System).
  • Approche centrée sur l'organisation (OCMAS, Organization Centred MAS).
ACMAS
  • S'occupe du comportement et de l'interaction des agents.
  • La structure du système est considérée comme formée à partir des interactions.
  • Approche Bottom-Up.
OCMAS
  • Prend l'organisation comme concept primitif.
  • Ne spécifie pas le comportement effectif des agents.
  • Organisation = structure d'un ensemble d'agents.
  • Approche Top-down.

Le modèle AGR

  • AGR signfie Agent, Groupe et Rôle.
  • Crée par Olivier Gutknecht et Jacques Ferber en 1998.
  • C'est un méta-model est non une méthodologie.
  • Permet de décrire un système conformément aux principes OCMAS.
  • Il est très générique.

Dans AGR :

  • Un agent est définie comme une entité autonome communiquante.
  • Il joue des rôles dans différents groupes.
  • Un groupe est un ensemble d'agents.
  • Un groupe peut être fondé par n’importe quel agent.
  • Un groupe peut être soit local soit distribué sur plusieurs machines.
  • Un rôle définie une fonction qu'un agent peut occuper au sein d'un groupe.
  • Chaque agent peut avoir plusieurs rôles.
  • Un même rôle peut être tenu par plusieurs agents.
  • Les rôles sont locaux aux groupes.
  • La tenue d’un rôle dans un groupe préexistant doit être demandée par l’agent et n’est pas forcément accordée.

Récap

Un rôle est caractèrisé par :

  • Unicité: un rôle est soit unique soit multiple.
  • Compétences: une condition que doit satisfaire l'agent pour pouvoir jouer un rôle.
  • Capacités: une propriété attribuée à un agent quand il joue un rôle particulier.
  • Un rôle par défaut est multiple, ne nécessite aucune compétence et n'offre aucune capacité.
  • L'agent qui crée un groupe se voit attribuer le rôle de gestionnaire de ce groupe.
  • Le gestionnaire peut accepter ou refuser l'admission d'un autre agent dans son groupe.
  • Il peut révoquer des rôles à des agents, ou les chasser de son groupe.

Quelques méthodes d'admission à un groupe :

  • Acceptation ou refus automatique.
  • Admission conditionnée par les compétences.
  • Admission contrainte par l’architecture.
  • Admission négociée.
  • Admission contrainte par l’état courant du groupe.

Notation

  • Un agent est représenté par une quille.
  • Un groupe par une ellipse entourant des agents.
  • Si un agent appartient à un groupe on peut définir son rôle à l'aide d'un héxagone ou par une description textuelle.

Travelling Salesman Problem

Ayant un ensemble de villes et la distance qui les sépare deux à deux.

On veut trouver le plus court chemin passant par toutes les villes.

Solution naïve : tester tout les chemins possibles.

Solution plus intelligente (heuristique) : ACO (Ant Colony Optimization).
